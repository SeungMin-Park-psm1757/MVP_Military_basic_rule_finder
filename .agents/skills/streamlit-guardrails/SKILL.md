# 목적
이 앱은 무료 티어 친화적이고, 법률자문 오인을 피하며, 근거 중심으로 답해야 한다.  
따라서 UI 단계에서 **질문 제한, 경고문, 시행일·출처 표시**를 확실히 유지해야 한다.

# 핵심 규칙
1. 기본 제한은 **일일 20회 질문**, **질문 1000자 제한**.
2. 남은 질문 수는 사이드바에서 항상 보여준다.
3. 답변 상단에 “실무 참고용, 법률자문 아님” 문구를 표시한다.
4. 근거 카드에는 최소한 아래를 보여준다.
   - 법령명
   - 자료유형
   - 조문번호/제목
   - 시행일
   - 원문 링크
5. 현재 조문과 과거 개정이력 자료유형이 섞여 보이지 않게 한다.
6. 컬렉션이 비어 있으면 사용자가 바로 실행할 명령어를 보여준다.

# 변경 전 점검
- quota 로직이 실제로 차감되는가?
- 글자 수 초과 시 즉시 차단되는가?
- 근거가 없을 때도 빈 화면이 아니라 보수적 안내를 하는가?
- 디버그 정보가 민감정보 없이 최소한으로 표시되는가?
