# 목적
이 프로젝트는 '멋있어 보이는 AI 챗봇'이 아니라, 군 행정의 **조직기억 단절 문제**를 해결하는 정책 제안이어야 한다.  
따라서 논문과 구현 문서를 함께 점검해 과장을 제거하고 설득력을 높인다.

# 레드팀 질문
1. 공개 법령 MVP 결과를 내부 군 예규 전체로 곧바로 일반화하고 있지 않은가?
2. '왜 바뀌었는가'를 개정이유 없이 추측하고 있지 않은가?
3. '어떻게 행동해야 하나'가 법률자문처럼 보이지 않는가?
4. 성과지표가 선언적 문구에 그치지 않고 측정 가능하게 정의되어 있는가?
5. 보안·권한·폐쇄망 요구사항을 별도 과제로 명시했는가?
6. MVP와 후속단계를 혼동하지 않는가?

# 수정 원칙
- 확대 가능성은 주장하되, 즉시 적용 가능성은 과장하지 않는다.
- 내부 규정 확장은 별도 보안·권한 체계를 전제로 둔다.
- 실무지원형 표현은 유지하되 법률자문 표현은 제거한다.
- “시간축 + 근거축 + 실무축” 설계가 논문과 코드 양쪽에 일관되게 반영되는지 확인한다.
