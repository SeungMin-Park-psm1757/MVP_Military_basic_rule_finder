# 릴리스 전 최종 점검 메모

## 1. 주요 리스크 점검 결과

### 치명 이슈
- 없음

### 중간 이슈
1. **법률자문 오인 가능성**
   - 조치: Streamlit 상단 안내문, Gemini 시스템 프롬프트, fallback 답변에 모두 “실무 참고용 / 법률자문 아님” 문구를 반영했다.
2. **개정이유 없는 사안의 과도한 추론 가능성**
   - 조치: 프롬프트와 README, 기획안에 “개정이유 문서가 있을 때만 직접 설명” 원칙을 명시했다.
3. **공개 법령 MVP를 내부 군 예규 전체로 과도 일반화할 위험**
   - 조치: 최종 기획안과 확장 계획 문서에서 내부 확장은 별도 보안·권한 체계 전제를 명시했다.

### 경미 이슈
1. **테스트 환경에서 Chroma 미설치 시 실행 불가**
   - 조치: JSON fallback store를 추가해 smoke test와 unit test가 동작하도록 보완했다.
2. **샘플 데이터가 실제 원문 전체가 아님**
   - 조치: `data/sample/README.md`와 README에 데모용 요약 샘플임을 명시했다.

## 2. 수정 후 최종 판단
- 논문 포지셔닝은 “법규 챗봇”보다 **조직기억 보존형 규정관리 시스템**으로 잡는 것이 타당하다.
- MVP 범위는 **A+B+C**로 적정하며, **D는 정책제안/후속과제**로 두는 것이 과장 방지에 유리하다.
- GitHub 공개 저장소 기준 구현은 공개 법률 데이터셋으로 충분히 재현 가능하다.
- 향후 내부 예규 확장은 기술보다 **보안·권한·운영 거버넌스**가 핵심 병목이다.
