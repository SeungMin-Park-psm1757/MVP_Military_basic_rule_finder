# 평가 프레임워크

## 1. 평가 목적
본 MVP의 평가는 “답변이 그럴듯한가?”가 아니라 다음 네 가지를 확인하는 데 목적이 있습니다.

1. 필요한 근거를 실제로 잘 찾는가
2. 개정이유를 과장 없이 설명하는가
3. 실무상 행동지침을 과도하게 단정하지 않는가
4. 기존 수작업보다 시간을 절감할 가능성이 있는가

## 2. 정량지표
| 지표 | 정의 | 측정방법 |
|---|---|---|
| Retrieval@K | 정답 근거가 상위 K개 검색 결과 안에 포함되는 비율 | 수작업 검증 |
| Citation Coverage | 답변 문장 중 근거 출처가 연결된 비율 | 문장단위 라벨링 |
| Unsupported Claim Rate | 근거 없이 단정한 문장 비율 | 레드팀 수기평가 |
| Time-to-Evidence | 사용자가 근거 문서를 찾는 데 걸리는 시간 | 시나리오 실험 |
| Daily Usage Control | 일일 사용량 제한이 정상 동작하는지 | 로그 점검 |

## 3. 정성지표
- 개정 배경 설명의 납득 가능성
- 실무 참고 답변의 명확성
- 상급자 보고용 근거 제시 적합성
- 민원성 반복질의 대응 유용성

## 4. 시험 시나리오 예시
1. 특정 조문 검색
2. 특정 개정의 이유 설명
3. 동일 주제의 과거/현재 비교
4. 실무처리 질문
5. 근거가 부족한 질문에 대한 보수적 응답 여부

## 5. 레드팀 체크리스트
- 법률자문처럼 오해될 문장이 있는가
- 개정이유가 명시되지 않았는데 추정을 사실처럼 말하는가
- 시행일과 공포일을 혼동하는가
- 현재 규정과 과거 규정을 뒤섞는가
- 출처 링크가 누락되었는가

## 6. 논문에 쓸 평가 문장 예시
> 본 시범체계의 평가는 단순 응답 만족도가 아니라, 규정 검토 업무에서 핵심적인 ‘근거 회수의 정확성’, ‘개정 취지 설명의 적정성’, ‘실무 참고 수준의 행동지침 제시 가능성’, ‘근거 탐색 시간 절감 가능성’을 중심으로 설계한다.
